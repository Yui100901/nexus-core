# nexus-core

## 项目定位

`nexus-core` 是一个面向个人软件、订阅制工具、物联网设备和私有运行环境的轻量级 License 认证与节点管理后端。

系统目标是提供一套简单、可扩展的后端能力，用于完成产品版本管理、License 授权、节点绑定、在线心跳、并发控制、节点监控以及服务端对节点的远程控制。

## 典型场景

- 个人软件通过 License Key 完成激活、续期和版本校验。
- 一套客户端程序或设备作为节点接入服务端，定期上报心跳。
- 服务端限制一个 License 可绑定的节点数量和在线并发数量。
- 服务端根据产品、版本、License 和节点状态判断客户端是否允许继续运行。
- 服务端向指定节点下发控制指令，节点根据自身实现执行对应功能。

## 核心概念

### 产品

产品表示一个可被授权和管理的软件、设备类型或业务系统。

产品需要支持：

- 创建、修改、删除。
- 启用、禁用或废弃。
- 维护产品版本。
- 设置最低支持版本。
- 关联可用功能或服务能力。

### 产品版本

产品版本表示产品的具体发布版本。

版本需要支持：

- 创建版本。
- 立即发布。
- 定时发布。
- 暂不发布。
- 启用、废弃。
- 通过版本号判断客户端版本是否仍被支持。

### License

License 表示一个授权凭证，通常以 License Key 的方式提供给客户端。

License 需要支持：

- 创建、查询、修改、删除。
- 激活、续期、吊销、恢复。
- 设置有效期。
- 设置可绑定节点数量。
- 设置最大在线并发数量。
- 设置授权产品范围。
- 设置可用功能范围。
- 记录激活、绑定、解绑、续期、吊销等操作日志。

### 节点

节点表示一个实际接入系统的运行实体，可以是一台设备、一个软件客户端、一个容器实例、一套用户运行环境，或任何具有唯一设备标识的客户端。

节点需要支持：

- 通过设备码唯一识别。
- 首次接入时动态创建。
- 绑定或解绑 License。
- 定期上报心跳。
- 上报运行环境信息。
- 维护在线、离线、封禁等状态。
- 支持黑名单或封禁机制。
- 记录节点操作日志。

### 节点控制服务

节点控制服务表示服务端可以对节点发起的远程控制能力。

节点控制不是固定的一组接口，而是基于服务定义和 Schema 的可扩展机制：

- 每个节点可以拥有自定义功能实现。
- 不同功能可以有不同的字段定义。
- 服务端将节点功能抽象为不同的控制服务。
- 每个控制服务可以定义自己的输入字段、输出字段和参数约束。
- 节点可以声明自己支持哪些服务，以及每个服务对应的 Schema。
- 服务端对节点通信时，需要根据节点预定义的 Schema 对控制请求进行转换。
- 转换后的请求由服务端发送给节点，节点按自身实现解析并执行。

示例：

- 节点 A 支持 `restart_process` 服务，字段包括 `process_name`、`delay_seconds`。
- 节点 B 支持 `set_config` 服务，字段包括 `config_key`、`config_value`、`reload`。
- 服务端调用控制服务时，不直接写死节点协议，而是先匹配节点 Schema，再生成节点可识别的消息。

## 功能需求

### 1. 认证与授权

- 支持 License Key 认证。
- 客户端注册时校验 License 是否存在。
- 校验 License 是否支持当前产品。
- 校验产品版本是否仍被支持。
- License 首次使用时可自动激活。
- License 过期、吊销、无效时拒绝访问。
- 后续可扩展用户登录能力，允许用户自助获取或管理 License。

### 2. 产品管理

- 产品创建、查询、更新、删除。
- 产品启用、禁用、废弃。
- 产品版本创建、发布、废弃。
- 支持立即发布和定时发布。
- 支持最低可用版本控制。
- 支持产品 Demo 或测试客户端完成认证与通信测试。

### 3. License 管理

- 单个 License 创建。
- 批量 License 创建。
- License 查询。
- License 修改。
- License 续期或缩短有效期。
- License 吊销和恢复。
- License 删除。
- 清理过期或无效 License。
- 设置最大绑定节点数。
- 设置最大在线并发数。
- 维护 License 与节点的绑定关系。

### 4. 节点管理

- 节点创建、查询、更新、删除。
- 根据设备码查找节点。
- 节点首次注册时自动创建。
- 节点绑定 License。
- 节点解绑 License。
- 清理无绑定节点。
- 维护节点状态，包括正常、离线、封禁。
- 支持节点运行环境元数据，例如系统版本、客户端版本、硬件信息、网络信息等。

### 5. 节点心跳与在线状态

- 节点定期发送心跳。
- 心跳时校验产品、版本、License、节点状态和绑定状态。
- 心跳成功后刷新节点在线状态。
- 超过心跳超时时间后标记节点离线。
- 支持按产品和 License 统计当前在线节点数。
- 支持按 License 进行并发限制。

### 6. 节点控制

- 服务端维护控制服务定义。
- 控制服务可以归属于产品，也可以作为通用服务复用。
- 控制服务包含名称、标识符、描述、输入 Schema、输出 Schema 和执行约束。
- 节点上报自己支持的控制服务列表。
- 节点上报每个服务对应的字段 Schema。
- 服务端保存节点的服务能力声明。
- 服务端发起控制指令时，根据节点 Schema 将标准控制请求转换为节点协议消息。
- 服务端记录控制指令的创建、发送、执行结果和失败原因。
- 控制指令需要支持同步返回和异步回执两种模式。
- 控制指令需要校验 License 权限和节点状态。
- 被封禁、离线、未绑定或 License 无效的节点不允许执行控制指令。

### 7. 监控与统计

- 统计产品使用情况。
- 统计 License 使用情况。
- 统计节点在线、离线和异常情况。
- 统计 License 当前并发数量。
- 记录异常访问、无效 License、过期 License、超出绑定数量、超出并发限制等事件。

### 8. 操作日志

- 记录产品变更日志。
- 记录版本发布日志。
- 记录 License 创建、激活、续期、吊销、删除日志。
- 记录节点绑定、解绑、封禁、解封日志。
- 记录节点控制指令发送和执行日志。

## 节点控制模型

节点控制建议拆分为以下几个模型。

### 控制服务定义

控制服务是服务端可管理的能力定义。

建议字段：

- `id`：服务 ID。
- `identifier`：服务唯一标识。
- `name`：服务名称。
- `description`：服务描述。
- `service_type`：服务类型，例如命令、配置、查询、动作。
- `input_schema`：输入字段 Schema。
- `output_schema`：输出字段 Schema。
- `enabled`：是否启用。

### 节点服务能力

节点服务能力表示某个节点实际支持哪些控制服务。

建议字段：

- `node_id`：节点 ID。
- `service_identifier`：服务标识。
- `schema`：该节点对此服务的实际字段定义。
- `protocol`：通信协议，例如 HTTP、MQTT、WebSocket。
- `endpoint`：节点侧接收地址或主题。
- `enabled`：是否启用。

### 控制指令

控制指令表示服务端对节点发起的一次调用。

建议字段：

- `id`：指令 ID。
- `node_id`：目标节点。
- `service_identifier`：目标服务。
- `payload`：服务端标准请求数据。
- `converted_payload`：根据节点 Schema 转换后的节点请求数据。
- `status`：待发送、已发送、执行中、成功、失败、超时。
- `result`：节点返回结果。
- `error_message`：失败原因。
- `created_at`：创建时间。
- `sent_at`：发送时间。
- `completed_at`：完成时间。

## 通信流程

### 客户端注册

1. 客户端携带设备码、License Key、产品 ID、版本号请求注册。
2. 服务端校验 License 是否存在。
3. 服务端校验 License 是否支持该产品。
4. 服务端校验产品版本是否被支持。
5. 服务端创建或更新节点。
6. 服务端绑定节点与 License。
7. 如果 License 未激活，则自动激活。

### 客户端心跳

1. 客户端定期上报设备码、License Key、产品 ID、版本号。
2. 服务端校验产品版本。
3. 服务端校验 License 状态。
4. 服务端校验节点状态。
5. 服务端校验绑定关系。
6. 服务端校验在线并发限制。
7. 校验通过后刷新在线状态。

### 服务端控制节点

1. 管理端选择目标节点和控制服务。
2. 服务端读取节点支持的服务 Schema。
3. 服务端校验输入字段。
4. 服务端将标准请求转换为节点协议消息。
5. 服务端发送消息至节点。
6. 节点执行对应功能。
7. 节点返回执行结果或异步回执。
8. 服务端记录控制日志和执行结果。

## 非目标

当前阶段不优先实现：

- 完整用户账号系统。
- 复杂租户体系。
- 支付系统。
- 完整前端管理后台。
- 跨服务分布式任务调度。

这些能力可以在核心 License、节点、心跳和控制链路稳定后继续扩展。

## 当前开发重点

建议优先完成以下主链路：

1. 产品和版本管理。
2. License 创建、激活、续期、吊销。
3. 节点注册、绑定和心跳。
4. License 绑定数量和并发限制。
5. 节点控制服务定义。
6. 节点 Schema 上报与保存。
7. 控制指令转换、发送和结果记录。

更具体的重构和实现推进清单见 [docs/requirements-checklist.md](docs/requirements-checklist.md)。

## 运行方式

默认配置文件为 `config-dev.yml`，默认使用 SQLite 数据库：

```yaml
db_path: ./data/test.db
```

启动服务：

```bash
go run .
```

默认端口：

```text
8080
```

Swagger 地址：

```text
http://localhost:8080/swagger/index.html
```

## 简易测试产品

仓库提供了一个最小测试客户端，用于快速验证服务端主链路。

先启动服务端：

```bash
go run .
```

再运行测试产品：

```bash
go run ./cmd/demo-product
```

它会自动执行以下流程：

1. 创建一个测试产品。
2. 创建并立即发布 `1.0.0` 版本。
3. 创建 License。
4. 使用 License 注册测试节点。
5. 连续发送心跳。

常用参数：

```bash
go run ./cmd/demo-product \
  -server http://localhost:8080 \
  -device demo-device-001 \
  -heartbeats 3 \
  -heartbeat-interval 1s
```

## 协议转换测试产品

仓库还提供了一个更完整的协议转换测试客户端，用于验证服务端控制链路的字段映射、类型转换、默认值、约束校验和输出转换。

先启动服务端：

```bash
go run .
```

再运行协议转换测试：

```bash
go run ./cmd/protocol-demo-product -server http://localhost:8080
```

它会自动执行以下流程：

1. 创建测试产品、版本、License。
2. 注册测试节点并发送心跳。
3. 创建 HTTP 控制服务和复杂节点 Schema。
4. 启动本地 HTTP 节点接收端，验证服务端转换后的 payload。
5. 构造非法 payload，验证转换约束会拒绝请求。
6. 创建 WebSocket 控制服务，模拟节点长连接，验证 WebSocket 下发消息转换。
7. 验证节点返回结果会按控制服务 `output_schema` 转换。

## 发布验证

发布前可以执行一键验证脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\release-validate.ps1 -Port 18081
```

该脚本会使用临时 SQLite 数据库和临时服务端口，依次执行全量测试、Race 检测、服务端与 Demo 构建、Swagger UI 冒烟、简易 Demo 端到端、协议转换 Demo 端到端、定时发布跨进程重启恢复、SQLite 同库重启演练，并在结束后恢复 `config-dev.yml`。

如需在具备真实 MQTT broker 的环境做配置冒烟，可传入：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\release-validate.ps1 -Port 18081 -MqttBrokerUrl tcp://127.0.0.1:1883
```

当前脚本会把 broker 配置注入临时服务；真实发布/订阅内容验收仍需要外部 MQTT 订阅端配合。

## 核心管理策略

- 产品删除采用保护策略：存在关联 License 或控制服务定义时拒绝删除；无依赖时会同时删除产品版本。
- 定时发布版本持久化在数据库中，服务启动后会扫描到期且未发布的版本并发布。
- 最低支持版本只能设置为已发布可用版本，当前最低支持版本不能直接废弃。
- 过期 License 可以通过正向续期恢复为可用；吊销 License 需要先恢复后才能续期。
- License 删除或无效清理会同步清理绑定和授权范围数据，并保留审计日志。
- 节点封禁支持记录原因，解封会清理封禁原因和封禁时间。
